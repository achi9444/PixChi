# PixChi 商業模式分析與功能規劃

## Context

用戶正在開發「拼豆圖紙生成工具」副業產品，已有完整核心功能（圖片轉換、調色板管理、Canvas 編輯、PDF 匯出、雲端草稿）。目標是建立工具＋創作者媒合市場。本文件整理商業模式建議、三種身分設計、浮水印機制、無金流方案，以及技術缺口。

## 二、浮水印運作機制

### 路人 PDF 匯出浮水印
- 每頁右下角固定小型 logo + "PixChi.app" 文字
- 不影響閱讀，但無法去除（作為推廣）
- 技術：在 `pdf-lib` 輸出時，根據 `role === 'guest'` 條件疊加 image/text layer

### 市場設計圖預覽浮水印
- 創作者上架的設計圖，市場頁顯示的縮圖/預覽 PDF 均有半透明浮水印
- 買家在外部平台付款後，創作者手動發送乾淨版本
- 技術：創作者上傳時，後端自動產生「浮水印預覽版」和「原始版」，分開儲存
  - 預覽版：public，供市場顯示
  - 原始版：private，創作者自行決定何時交付

### 設計圖授權說明
- 每個設計圖可標示「個人使用」或「商業授權」
- 價格差異由創作者自訂，在市場列表顯示
- 授權說明只是文字標籤，實際交易在外部平台完成

## 三、無金流方案（媒合平台）

### 運作流程
1. 路人/會員 → 瀏覽市場，看到創作者作品
2. 點擊「聯絡創作者」→ 跳轉到創作者設定的外部連結（賣貨便店面、好賣＋頁面、Line ID 等）
3. 交易全程在外部平台完成，PixChi 不介入
4. 平台不收交易抽成，無需金流資質，無糾紛責任

### 創作者主頁可設定
- 外部連結（賣貨便 / 好賣＋ / 其他）
- 接單狀態（開放中 / 暫停接單）
- 擅長風格 tag
- 作品集（已上架設計圖）

### 優點
- 技術複雜度低，前期成本小
- 不需要金流申請、不需要處理退款/糾紛
- 法律風險低（只是廣告板 + 工具）

## 四、收入模式

### 主要收入：創作者訂閱
- 月費制（建議 NT$199–299/月 或 NT$1,999/年）
- 訂閱解鎖：市場上架權限、PDF 匯入、大圖分塊模式、報價工具、施工模板
- 沒有訂閱 = 無法在市場出現 = 無法接客

### 次要收入（未來可考慮）
- 推薦版面費：創作者付費在市場首頁曝光
- 廣告/贊助：特定調色板品牌（拼豆廠商）付費推廣
- 會員升級：若會員需要更多雲端草稿，可考慮付費方案

## 五、目前技術缺口（需補實的功能）

### P0 必須有（MVP 商業模式缺口）
1. **註冊流程** — 目前只有登入，沒有 signup（`/api/auth/register` 不存在）
2. **創作者訂閱管理** — 後端需要訂閱狀態欄位、前端需要升級引導
3. **市場頁面** — 設計圖列表、搜尋/篩選
4. **創作者主頁** — 個人資料、作品集、外部連結設定

### P1 核心商業功能
5. **設計圖上架系統** — 創作者上傳、標題/說明/標籤/授權類型/預覽圖
6. **浮水印產生** — Guest PDF 匯出時自動加水印
7. **PDF 圖紙匯入還原** — 創作者功能（目前只有 UI，未接後端）
8. **詢問/聯絡按鈕** — 跳轉到創作者外部平台

### P2 完善體驗
9. **收藏功能** — 會員收藏設計圖
10. **設計圖標籤/分類** — 角色、難度、尺寸、風格
11. **創作者認證標章** — 訂閱中的創作者顯示標章
12. **設計圖排序/推薦** — 熱門、最新、收藏數

## 七、建議開發順序

1. 補上**註冊流程**（最基礎缺口）
2. 設計 DB Schema：`designs`（設計圖）、`creator_profiles`、`subscriptions` 表
3. 實作 Guest PDF 浮水印（差異化核心）
4. 市場頁面 MVP（僅列表 + 聯絡按鈕）
5. 創作者主頁設定
6. 設計圖上架
7. 訂閱方案 + 付款（可先用 LinePay/ECPay 或直接找 Stripe Taiwan）

## 商業模式總結

```
路人 → 免費使用工具 (有浮水印) → 看到市場 → 引導加入會員
會員 → 免費/低門檻 → 使用工具 + 聯絡創作者 → 平台黏著度
創作者 → 月費訂閱 → 在市場曝光 → 接客訂單 → 外部平台交易
平台收入 = 創作者訂閱費（+ 未來推薦版面/廣告）
```

這個模式輕、快、合法，前期成本低，可以先驗證市場，等規模到了再評估是否加入金流。
