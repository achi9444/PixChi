# PixChi MVP Spec (v0.1)

## 1. 產品目標
- 將任意上傳圖片轉換為拼豆格線圖，並維持穩定的顏色對應結果。
- 輸出結果必須可直接用於報價與生產。
- 色號統計必須完整（所有使用到的色號），不得只顯示 top-N。

## 2. 核心流程
1. 建立專案
2. 上傳圖片
3. 設定格線（cols x rows）
4. 選擇版面模式：`fit` / `lock` / `pad`
5. 轉換為格線圖
6. 手動調整（單格修改 / 區域替換）
7. 匯出圖紙 + 材料清單 + 報價資料

## 3. 畫布與比例規則
- 每個格子固定為正方形拼豆。
- 格子寬度 = image_width / cols
- 格子高度 = image_height / rows
- 模式：
  - `fit`：保持原圖比例，將 rows/cols 調整到最接近的有效比例。
  - `lock`：強制使用指定 rows/cols，必要時裁切原圖。
  - `pad`：強制使用指定 rows/cols，必要時補透明/白色邊界。
- 格線上限：
  - Free tier：最大 64x64
  - Pro tier：最大 128x128

## 4. 顏色對應規則
- 每格來源色：使用 median RGB（預設）降低雜訊影響。
- 將顏色轉換至 Lab。
- 以 DeltaE2000 比對最接近色庫顏色（預設）。
- 可選比對策略：
  - `lab_nearest`
  - `rgb_nearest`
  - `palette_priority`（品牌優先）
- 可選後處理：
  - 降噪強度
  - 併色閾值

## 5. 必要的完整色號統計
- 結果中必須列出所有使用到的色號。
- 每個色號必備欄位：
  - 色號（name）
  - hex
  - 數量（拼豆顆數）
  - 佔比（%）
  - 預估材料成本（若有價格表則輸出）
- 必備總計欄位：
  - 總顆數
  - 總色號數
  - 預估總成本
- UI 可提供搜尋/篩選，但匯出必須維持完整清單。

## 6. UI 結構
- 左側面板：轉換設定
  - Basic（預設展開）：上傳、格線尺寸、保持比例、群組、轉換模式
  - Advanced（預設收合）：denoise、併色閾值、顯示切換
- 中央：可編輯的格線預覽
- 右側：完整色號統計 + 材料清單 + 報價區塊
- 底部動作列：
  - 匯出 Pattern PDF
  - 匯出 Material CSV
  - 匯出 Quote Sheet

## 7. 專案與群組行為
- 預設作用群組：`小舞`
- 使用者未切換群組前，所有新增色號都寫入作用群組。
- 同一群組內色號名稱不可重複。

## 8. 匯出項目
- Pattern PDF：
  - 含座標的格線圖
  - 色號圖例
  - 完整色號統計區塊
- Material CSV：
  - group_name,color_name,hex,count,ratio,unit_cost,line_total
- Quote Sheet：
  - totals + labor + margin + final price

## 9. 非功能性目標
- 轉換目標時間（64x64）：<= 2s（桌機基準）
- 相同參數必須得到可重現結果（deterministic）。
- 手動編輯需支援 undo/redo（至少 20 步）。

## 10. MVP 驗收清單
- [ ] 可將圖片轉為拼豆格線圖
- [ ] 支援 fit/lock/pad
- [ ] 支援手動修色
- [ ] 色號統計可完整顯示與匯出
- [ ] 可匯出 PDF + CSV
- [ ] 可輸出可報價的總計資料
